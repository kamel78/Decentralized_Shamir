How It Works

Our Enhanced Shamir Secret Sharing solution provides a secure and robust framework for distributing cryptographic keys among multiple participants.

Commission Creation

Admin creates a commission with a unique ID, specifying the number of participants (n) and the threshold (t).

User Assignment

Registered clients are added to the commission by the administrator.

Key Distribution

The cryptographic key is split into n shares and distributed to commission members.

Secure Reconstruction

When needed, at least t participants must combine their shares to reconstruct the original key.

Enhanced Shamir Secret Sharing Explained

A comprehensive overview of this advanced cryptographic technique

What is Enhanced Shamir Secret Sharing?

Enhanced Shamir Secret Sharing builds upon Adi Shamir's original 1979 algorithm by incorporating elliptic curve cryptography. This enhancement provides a robust public/private key infrastructure while maintaining the core threshold sharing principle.

Key Improvements:

  • Eliminates dependency on a central authority
  • Optimized for distributed systems
  • Provides both public and private key components
  • Enhanced security through elliptic curve cryptography
  • More resistant to quantum computing attacks
🔐

How Enhanced Shamir Works

  1. User Setup: n participants each generate their own secret shares and exchange subshares securely.
  2. Public Key Generation: The system combines contributions from all participants to create a public key without revealing any individual's private information.
  3. Secret Distribution: Each participant receives their share of the secret key, with no single party having access to the complete key.
  4. Threshold Reconstruction: When at least t participants combine their shares, the original secret key can be reconstructed for use.

Traditional Shamir

  • Central authority distributes shares
  • Single point of failure
  • No built-in public key system

Enhanced Shamir (Our Solution)

  • Distributed authority model
  • Public/private key infrastructure
  • Efficient elliptic curve operations
  • Resistant to collusion attacks

Application Workflow

Our platform supports two main user roles with distinct workflows:

Administrator

  1. Creates a commission with n members and threshold t
  2. Publishes marche events for public users
  3. Manages commission members
  4. Computes public key and reconstructs secret to decrypt documents

Public Users

  1. View published marche events
  2. Submit encrypted offers (documents)
  3. Offers remain encrypted until commission members reconstruct the secret
